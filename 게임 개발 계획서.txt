게임 개발 계획서

1. 개요
본 문서는 제공된 요구사항에 따라 게임 프로젝트를 개발하기 위한 상세 계획을 기술합니다. 기존 'LabProject08-7'을 기반으로 UI, 게임 상태 관리, 고급 렌더링, 게임 시스템 등 새로운 기능들을 체계적으로 구현하는 것을 목표로 합니다.

2. 기반 프로젝트
- LabProject08-7

3. 상세 개발 계획

가. 1단계: 게임 상태 관리 및 UI 시스템 기반 구축

- 목표: 게임의 기본 흐름(시작 화면, 게임 중, 일시정지)을 제어할 수 있는 상태 머신을 구현하고, 텍스처 기반의 UI 렌더링 시스템을 구축합니다.

- 세부 계획:
  1. 게임 상태 관리 시스템 정의:
     - `eGameState` 열거형으로 `StartScreen`, `InGame`, `PausedMenu` 상태 정의
     - `CGameFramework`에 현재 게임 상태를 저장하고 제어하는 로직 추가
     - 메인 루프(`FrameAdvance`)에 상태별 `Update` 및 `Render` 함수 분기 처리

  2. 텍스처 아틀라스 기반 UI 렌더링 시스템:
     - 알파벳 텍스처 아틀라스(`FontAtlas.dds`)를 사용하는 `CSpriteFont` 클래스 구현
     - `RenderText(string, x, y)` 함수를 통해 문자열을 화면에 렌더링하는 기능 구현
     - UI 버튼(`CUIButton`) 클래스를 설계하여 텍스트, 위치, 크기 및 클릭 이벤트 처리

  3. 상태별 화면 및 상호작용 구현:
     - **시작 화면:** "START" 버튼 렌더링 및 클릭 시 `InGame` 상태로 전환
     - **게임 중:** `ESC` 키 입력 시 `PausedMenu` 상태로 전환
     - **일시정지 메뉴:** "RESUME", "QUIT" 버튼 렌더링. "RESUME" 클릭 시 `InGame` 상태로, "QUIT" 클릭 시 게임 종료

나. 2단계: 게임 월드 및 고급 렌더링 구현

- 목표: 모델, 지형, 특수 효과를 활용하여 게임 월드를 구성하고, 사실적인 렌더링 효과를 추가합니다.

- 세부 계획:
  1. 게임 월드 구성:
     - 요구사항에 명시된 헬리콥터, 탱크, 건물 등의 3D 모델을 Placeholder 형태로 제작하여 배치 (모델링 제약으로 인해 간단한 형태로 구현)
     - 기존 지형 시스템을 확장하여 물, 나무, 풀 등을 빌보드 기법으로 구현 (기하 셰이더 활용)

  2. 고급 렌더링 기술 적용:
     - **노멀 맵핑:** 모델의 법선 맵을 사용하여 입체감과 사실감을 더하는 조명 계산 구현
     - **반사 효과:** 건물 유리창 또는 벽면에 큐브맵을 이용한 환경 맵핑을 적용하여 주변 환경이 반사되는 효과 구현

다. 3단계: 플레이어 및 게임 시스템 완성

- 목표: 플레이어의 세부 동작을 구현하고, 게임의 핵심 시스템을 완성합니다.

- 세부 계획:
  1. 플레이어 구현:
     - 헬리콥터, 탱크, 자동차 중 하나를 플레이어로 선택 (헬리콥터 우선 구현)
     - 1인칭 시점에서 플레이어 모델이 반투명하게 보이도록 렌더링 상태(Blending) 조절

  2. 핵심 게임 시스템:
     - **충돌 검사:** 플레이어, 적, 지형, 건물 간의 충돌을 판정하고 처리하는 로직 구현
     - **절두체 컬링:** 카메라의 시야각 밖에 있는 오브젝트를 렌더링하지 않도록 하여 성능 최적화
     - **특수 효과:** 간단한 파티클 시스템을 이용한 폭발 효과 구현

  3. 최종 UI 적용:
     - 게임 상태(점수, 적 위치 등)를 2단계에서 구현한 UI 시스템을 통해 화면에 표시
